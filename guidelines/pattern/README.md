# SvS UI (Nova) - Patterns

This directory contains detailed documentation for common patterns and user flows in the Svenska Spel Design System.

Pattern documentation covers user experience patterns, interaction flows, and best practices for common scenarios like navigation, login, error handling, and forms.

---

## Available Pattern Topics

- [Navigation](./Navigation.md) - Navigation principles, hierarchy, and patterns for guiding users through digital environments
- [Login](./Login.md) - Authentication patterns and login flows for secure user access
- [Errors](./Errors.md) - Error patterns and messaging for helping users understand and resolve issues
- [Form](./Form.md) - ⚠️ **WIP** - Form patterns, validation, and best practices (coming soon)

---

## Adding New Pattern Topics

When importing a new pattern topic from Figma:

1. Create a new `.md` file named after the topic (e.g., `Checkout.md`, `Onboarding.md`)
2. Follow the established documentation structure
3. Add a link to the topic in this README
4. Add a link to the topic in the main `Guidelines.md` file
