# Icons and Illustrations

Our icons and illustrations have the main purpose of making things easier. But for us, they are equally important proof that we are in the entertainment industry, so the subjects should be entertaining and unexpected.

---

## Philosophy

**Making Things Easier with Personality**

Icons and illustrations serve both functional and expressive purposes in our visual language.

**Core Principles:**
- **Simple and clear style** - Easy to understand at a glance
- **Different levels** - To cover all needs from functional to emotional
- **Multi-channel use** - Works in both digital and analog communication
- **Animation adds life** - Motion brings personality and engagement

**Entertainment Industry:**
Our icons and illustrations show we belong in the entertainment industry - they have a twinkle in the eye and may surprise and invite smiles.

---

## Icon and Illustration Hierarchy

We use three distinct levels of visual elements, each serving specific purposes.

### 1. Enkla Ikoner (Simple Icons)

**Purpose:**
Simple, functional icons that make interfaces clearer and easier to navigate.

**Characteristics:**
- **Simple expression** - Minimal detail, maximum clarity
- **Clear and self-explanatory** - Can often stand alone without text
- **Consistent style** - Works across all our brands
- **Digital-first** - Primarily created for digital channels

**Use Cases:**
- User interface elements
- Navigation
- Action indicators
- Feature representation
- Status icons

**Key Principle:**
Functional icons are simpler and sometimes used without text, which requires them to be clear and self-explanatory.

**Cross-Channel:**
While primarily created for digital channels, they can also be used in other communication.

**Link to Foundation:**
For technical implementation details, see the [Iconography section under Foundation](../foundation/Iconography.md).

---

### 2. Piktogram (Pictograms)

**Purpose:**
Pictograms should not be confused with icons. They have greater creative freedom and are used to quickly convey messages and simplify complex ideas.

**Characteristics:**
- **Illustrative approach** - More detailed than simple icons
- **Larger format** - Used only in bigger sizes, allowing more detail
- **Freer grid** - Drawn in a more flexible grid system
- **Information visualization** - Explains concepts visually

**Use Cases:**
- Visualizing complex concepts
- Information graphics
- Service explanations
- Process diagrams
- Educational content

**Purpose:**
Our illustrative pictograms visualize information in a simple and clear way. Their purpose is to make information more playful and engaging.

**Size Consideration:**
Because they are only used in larger formats, they can be more detailed than simple icons.

---

### 3. Illustrationer (Illustrations)

**Purpose:**
Emotional, character-driven illustrations that warm up communication and create connection.

**Characteristics:**
- **Unique style** - Created by illustrator Falko Grentrup
- **Energetic and cheerful** - Slightly rounded and happy expression
- **Perspective and shapes** - Dynamic viewpoints
- **More emotional** - Than our pictograms

**Artist:**
Drawn by illustrator **Falko Grentrup**, giving them a distinctive and consistent style.

**Emotional Purpose:**
More emotional than our pictograms, illustrations can:
- Warm up dialogs on sensitive topics
- Add personality to error messages
- Create empathy in user interactions
- Build relationships through visual storytelling

**Use Cases:**

**Sensitive Topics:**
- Responsible gaming (Spelkoll)
- Account limitations
- Support and guidance
- Error messages

**Relationship Building:**
Works well in relationship-building contexts when topics are connected to Svenska Spel:
- Welcome messages
- Onboarding
- Community features
- Personal milestones

**Dialog Enhancement:**
Can warm up dialogs on sensitive topics like responsible gaming, or be used in a dialog box when something has gone wrong.

**New Motifs:**
New subjects may be added in batches over time. Find all current illustrations [in the asset library].

## Simple Icons (Enkla Ikoner)

### Style Guidelines

**Visual Characteristics:**
- Clean lines
- Minimal details
- Consistent stroke weight
- Rounded corners where appropriate
- Optical balance

**Grid System:**
- Built on strict grid for consistency
- Pixel-perfect alignment
- Standard sizing (16px, 24px, 32px, 48px)

**Color:**
- Primarily monochrome (black or white)
- Adapt to context color
- Semantic colors for status (success, warning, error)

---

### Usage

**With Text:**
- Icon + label for clarity
- Appropriate spacing
- Aligned consistently

**Without Text:**
- Only for universally recognized actions
- Must be completely self-explanatory
- Test with users when in doubt

**Sizing:**
- **16px** - Inline with text, tight spaces
- **24px** - Standard UI elements
- **32px** - Prominent actions
- **48px+** - Large touch targets, hero elements

---

### Accessibility

**Touch Targets:**
- Minimum 44x44px for interactive icons
- Adequate spacing between clickable icons
- Clear visual feedback on interaction

**Contrast:**
- Ensure sufficient contrast against background
- Test in both light and dark modes
- Don't rely solely on color to convey meaning

**Screen Readers:**
- Always provide text alternatives
- Descriptive labels for icon-only buttons
- ARIA labels where appropriate

---

## Creating New Icons and Illustrations

### Simple Icons

**Process:**
1. Identify functional need
2. Research universal understanding
3. Sketch concepts
4. Refine on grid
5. Test for clarity
6. Optimize for multiple sizes

**Guidelines:**
- Follow existing icon system
- Maintain consistent style
- Ensure optical balance
- Test at intended sizes

---

### Pictograms

**Process:**
1. Identify concept to visualize
2. Break down into key elements
3. Sketch illustrative approach
4. Refine details
5. Ensure clarity at intended size
6. Test comprehension

**Guidelines:**
- More creative than icons
- Still clear and purposeful
- Consistent with existing set
- Appropriate level of detail

---

### Illustrations

**Artist Collaboration:**
New illustrations are created by Falko Grentrup to maintain stylistic consistency.

**Process:**
1. Identify need and context
2. Brief artist on purpose and emotion
3. Review sketches
4. Refine to match brand
5. Prepare for implementation

**Consistency:**
All illustrations should match the established style to maintain brand coherence.

## Technical Specifications

### File Formats

**Simple Icons:**
- **SVG** - Preferred for digital use, scalable
- **PNG** - Multiple sizes for specific needs
- **Icon fonts** - For system-wide consistency

**Pictograms:**
- **SVG** - Primary format for scalability
- **PNG** - High resolution for print

**Illustrations:**
- **PNG** - Standard digital use
- **SVG** - When possible for web
- **High-res files** - For print applications

---

### Sizing

**Simple Icons:**
- 16px, 24px, 32px, 48px (standard sizes)
- Always use even numbers
- Pixel-perfect at base sizes

**Pictograms:**
- Minimum 64px
- Scale to context
- Maintain aspect ratio

**Illustrations:**
- Context-dependent sizing
- Maintain detail at smaller sizes
- Test visibility on mobile

---

### Color

**Simple Icons:**
- Inherit text color by default
- Semantic colors for status
- Monochrome in most cases

**Pictograms:**
- Brand colors allowed
- Multi-color for clarity
- Consistent color usage

**Illustrations:**
- Full color palette
- Warm, inviting tones
- Consistent with brand colors

---

## Best Practices

### Simple Icons

**✓ Do:**
- Keep simple and recognizable
- Use consistent stroke weight
- Align to pixel grid
- Test at intended sizes
- Provide text labels when possible

**✗ Don't:**
- Over-complicate designs
- Use too many different styles
- Rely solely on icon without text
- Ignore optical balance
- Use inconsistent sizing

---

### Pictograms

**✓ Do:**
- Visualize complex concepts clearly
- Use appropriate level of detail
- Maintain consistent style
- Size appropriately for context
- Make information engaging

**✗ Don't:**
- Confuse with simple icons
- Use in very small sizes
- Over-detail
- Ignore brand consistency
- Replace all text with images

---

### Illustrations

**✓ Do:**
- Use for emotional connection
- Match Falko Grentrup style
- Warm up sensitive topics
- Build relationships
- Add personality to interfaces

**✗ Don't:**
- Use for purely functional purposes
- Create new styles inconsistently
- Overuse in one interface
- Replace clear communication
- Use when simple icon would suffice

---

## Accessibility Considerations

### All Visual Elements

**Color Independence:**
- Don't rely on color alone
- Include shapes, labels, or context
- Test in grayscale

**Contrast:**
- Sufficient contrast ratios
- Works in light and dark modes
- Visible to color-blind users

**Text Alternatives:**
- Alt text for images
- ARIA labels for icons
- Descriptive captions

**Cognitive Load:**
- Clear and purposeful
- Not overly complex
- Supports understanding
