# Design Elements

A unified expression of design elements for all of Svenska Spel. The elements are primarily designed to work in digital customer interactions and then applied according to the same principles for other channels.

---

## Philosophy

Design elements provide visual tools to reinforce messages, highlight information, and guide users through our digital and physical experiences.

**Core Principles:**
- **Simple expression** - Designed to fit our entire offering
- **Primarily red** - Uses our brand red colors
- **Three variants** - Word boxes, price labels, and tags
- **Digital-first** - Optimized for digital customer interactions
- **Cohesive system** - Unified graphic expression with rounded corners

---

## Element Types

Our design elements come in three distinct types, each serving specific purposes.

### 1. Ordlådor (Word Boxes)

**Purpose:**
Word boxes are used to reinforce or clarify a message. They are set in Svenska Spel Arena, which gives them a powerful feeling.

**Typography:**
- Set in **Svenska Spel Arena Pro Bold**
- Uppercase text
- Creates strong visual impact

**Usage:**
- Reinforce key messages
- Clarify important information
- Create visual hierarchy
- Add emphasis to headlines

**Combinations:**
Word boxes can be combined for more expressions:
- **Two word boxes in different colors** can be placed together to create contrast and rhythm
- **Two words can be set together** in the same word box

---

### 2. Prislappar (Price Labels)

**Purpose:**
Price labels relate to word boxes and come in two variants for highlighting prices, offers, and directional information.

**Variants:**

**Arrow on One Side:**
- Pointer shape on one edge
- Can be flipped to point in different directions
- Useful for directional cues

**Arrow on Both Sides:**
- Pointer shapes on both edges
- Creates symmetrical emphasis
- Works well centered in layouts

**Flexibility:**
The shapes can naturally be rotated if it suits the context.

**Relationship to Word Boxes:**
Price labels are similar to word boxes but have slightly less colored area at the top and bottom to balance the pointed edges.

---

### 3. Taggar (Tags / Badges)

**Purpose:**
Tags function as support in navigation and can also serve as prompts for interaction. They have a clear connection to the digital customer journey and UI.

**Digital Badges:**
Tags are primarily used in digital interfaces to:
- Support navigation
- Indicate status
- Prompt user interaction
- Label content categories
- Highlight new features

**Typography:**
- Written in uppercase (versaler)
- Can be filled or outlined

**Variants:**

**Filled:**
- Solid background color
- High emphasis
- Strong visual weight

**Tonal:**
- Subtle background tint
- Lower emphasis
- Gentler visual presence

**Color Variants:**
Available in six semantic color variants:
- **Brand** - Primary Svenska Spel red
- **Contrast** - High contrast variant
- **Neutral** - Gray/beige tones
- **Success** - Green for positive states
- **Attention** - Yellow/orange for warnings
- **Important** - Red for critical information

## Creating Design Elements

Design elements have a cohesive graphic expression with rounded corners and can be modified through different color choices and degrees of rotation. The exact appearance is determined based on the design system.

### Construction Principles

**Word Boxes:**

**Color Area:**
The colored area of a word box corresponds to a hyphen (-) in the text size, with the same measurements on all sides.

**Connecting Word Boxes:**
When two word boxes are placed at the same height level, they are joined with a concave corner.

**Spacing:**
- Consistent padding around text
- Proportional to text size
- Creates balanced appearance

---

**Price Labels:**

**Balance:**
Price labels are similar to word boxes but have slightly less colored area at the top and bottom to balance the pointed edges.

**Directional Use:**
- Arrow points can indicate direction
- Can point left, right, or both
- Rotation allows for various orientations

---

**Tags (Badges):**

**Pill Shape:**
- Fully rounded ends
- Compact horizontal design
- Minimal padding for tight spaces

**Size Variants:**
- Small - Compact spaces
- Medium - Standard usage
- Large - Prominent placement

---

## Color Usage

Design elements can be colored in the brand's three red shades or used with red text on a white background.

### Red Color Variants

**Primary Red:**
- **HEX:** `#ED0000`
- **Usage:** Most common, high energy
- **Context:** Primary actions, key messages

**Dark Red 1:**
- **HEX:** `#A20020`
- **Usage:** Contrast and depth
- **Context:** Secondary emphasis, layering

**Dark Red 2:**
- **HEX:** `#62001D`
- **Usage:** Deepest shade, strong contrast
- **Context:** Dark backgrounds, maximum impact

### Inverted Colors

**Red Text on White:**
- White background with red text
- Lower visual weight
- Suitable for less emphasis
- Works well in clusters

**Accessibility:**
Always ensure sufficient contrast between text and background colors.

---

## Rotation and Orientation

Word boxes and price labels can be rotated if it suits the context.

### Rotation Guidelines

**Subtle Rotation:**
- Small angles (5-15 degrees) add dynamism
- Creates energy and movement
- Maintains readability

**Extreme Rotation:**
- Avoid rotating beyond 30 degrees
- Text must remain readable
- Consider user experience

**Purpose-Driven:**
- Rotate to follow visual flow
- Match composition dynamics
- Create visual interest

**Tags:**
Generally should not be rotated - maintain horizontal orientation for UI consistency.

---

## Usage Guidelines

### Word Boxes

**✓ Do:**
- Use to reinforce key messages
- Set in Svenska Spel Arena uppercase
- Combine boxes for contrast
- Use rounded corners consistently
- Apply appropriate padding

**✗ Don't:**
- Use for body text
- Make boxes too small to read
- Over-rotate (beyond readability)
- Mix with non-brand fonts
- Use on busy backgrounds without contrast

---

### Price Labels

**✓ Do:**
- Use for pricing and offers
- Point arrows directionally when appropriate
- Balance with surrounding content
- Rotate to suit layout

**✗ Don't:**
- Use as general text containers
- Make arrows point randomly
- Overcomplicate with multiple directions
- Use when word box would suffice

---

### Tags (Badges)

**✓ Do:**
- Use for navigation support
- Indicate status clearly
- Keep text concise (1-2 words)
- Choose appropriate color variant
- Maintain uppercase styling

**✗ Don't:**
- Use for long text strings
- Mix filled and outlined styles inconsistently
- Rotate or skew badges
- Use as primary navigation
- Over-saturate interface with tags

## Technical Specifications

### Border Radius

**Word Boxes & Price Labels:**
- Rounded corners proportional to element size
- Typically 4-8px for digital
- Scale appropriately for print

**Tags:**
- Fully rounded ends (pill shape)
- Border radius = 50% of height
- Creates distinctive appearance

---

### Padding and Spacing

**Word Boxes:**
- Padding equals hyphen (-) width at text size
- Same measurement on all sides
- Proportional to text

**Price Labels:**
- Slightly less padding top/bottom
- Balances pointed edges
- Maintains visual weight

**Tags:**
- Minimal horizontal padding
- Vertical padding for comfortable reading
- Compact but not cramped

---

### File Formats

**Vector (SVG):**
- Preferred for digital use
- Scalable without quality loss
- Supports color variants

**Raster (PNG):**
- For specific implementations
- Transparent background
- Multiple resolutions

**Design Files:**
- Adobe Illustrator (.ai)
- Figma components
- Sketch symbols

## Accessibility Considerations

### Color Contrast

**Text Readability:**
- White text on red: Verify WCAG AA compliance
- Red text on white: Ensure sufficient contrast
- Test with color blindness simulators

**Semantic Colors:**
- Don't rely on color alone to convey meaning
- Include icons or text labels
- Provide text alternatives

---

### Size and Touch Targets

**Minimum Sizes:**
- Tags: Minimum 24px height for touch targets
- Word boxes: Large enough for readability
- Price labels: Sufficient for text legibility

**Interactive Elements:**
- Minimum 44x44px touch target
- Adequate spacing between elements
- Clear visual feedback on interaction
